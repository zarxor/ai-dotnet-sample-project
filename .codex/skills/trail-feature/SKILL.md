# Trail Feature Skill

Use this skill when adding or refining a TrailForge Planner product feature.

## Goal

Add a small, visible hiking-planner capability that is easy to demonstrate, explain, and review.

## Process

1. Identify the user need in hiking-planning terms.
2. Update `Program.cs` if the feature needs new sample data, an API endpoint, or a changed response shape.
3. Update `wwwroot/app.js` for rendering or interaction.
4. Update `wwwroot/styles.css` for layout, state, or responsive behavior.
5. Keep the feature dependency-free unless the existing code clearly needs a library.

## Quality Bar

- The feature should be visible in the browser.
- The feature should use concrete trail-planning language.
- The code should stay small enough to explain during a live demo.
- Any new UI control should have a label or accessible name.
